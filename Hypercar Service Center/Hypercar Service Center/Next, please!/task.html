Description

When a customer gets to the front of the queue, the operator needs to click the "Process next" button so work can begin. This should happen when one of the mechanics has finished the previous job and is ready to serve the next customer. You can't define the exact time when this will occur in the application, so you need to leave the task to the operator.

When the "Process next" button is pushed, the application selects the next customer according to priority defined by the algorithm:

  1. If there are any customers in the oil change queue, they are served first.
  2. When the oil change queue is empty, customers in the tire inflation queue are next in line.
  3. Customers requiring diagnostics are served last.

As soon as the operator presses the button, the selected ticket number should be displayed, and the relevant queue's length should be reduced by one. The ticket doesn't need to be saved once this action has been completed â€” its number just needs to be shown on the screen.

Objectives

Your final task is to make a POST handler for processing the next ticket. After the request is received, the appropriate ticket should be removed from the queue. The handler should process POST requests at "<base_url>/processing", where <base_url> is the host.

Don't forget that there also needs to be a handler to show information about the next ticket.

You need to create a <div> element with the content "Next ticket #<number_of_ticket>" on the page that will be displayed to all customers:

<div>Next ticket #7</div>

If there are no tickets to process, the content of the element should be "Waiting for the next client":

<div>Waiting for the next client</div>

The page with the next ticket should be available at "<base_url>/next", where <base_url> is the host.

Be cautious: URLs in the urls.py module should not contain slashes at the end.

Example

Assuming all the mechanics are busy and the situation is as a follows:

  • Customers with tickets 5 and 8 are waiting in the oil change queue.
  • The customer with ticket 4 is waiting for tire inflation.
  • The customer with ticket 3 is waiting for diagnostics.

When the mechanics finish the current job, and the operator pushes the "Process next" button, the customer screen will display the following information:

Next ticket #5

The next ticket is 5 because customers requiring oil changes go first. The ticket after 5 will be 8, followed by 4, and the last ticket will be 3.
