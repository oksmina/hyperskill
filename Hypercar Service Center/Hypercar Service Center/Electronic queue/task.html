Description

There are two or three hard-working mechanics based in the service center each day, and they deal with clients in the order they arrive. If several arrive at the same time, the line of waiting customers can become really long. Usually, diagnostic tests take 30 minutes, inflating tires takes 5 minutes, and oil changes take about 2 minutes.

The company conducted a survey and discovered that customers are prepared to wait in the queue for more time if the service they require takes longer. To reflect this, a new processing algorithm now needs to be implemented:

  1. Any customers waiting in line for an oil change should be served first.
  2. Those who want their tires inflated should be dealt with next.
  3. Customers who require a diagnostics check should be served last.

Tickets should be given a reference number so they can be distinguished from each other. The first ticket issued should be number 1, the second should be number 2, and so on. Not all customers are prepared to wait for a long time, so the system also needs to display an estimate of how long it will be until their turn. Don't forget that the line is processed in a particular order!

Objectives

Your task is to create a handler that gives tickets to the clients. The handler should serve GET requests on the following URLs:

  • <base_url>/get_ticket/change_oil
  • <base_url>/get_ticket/inflate_tires
  • <base_url>/get_ticket/diagnostic

Each page should contain a <div> element with the content "Your number is <ticket_number>", where <ticket_number> is the ticket's reference number. To calculate the estimated time, you should sum the time it will take to process all the tickets that will be worked on before the current one, according to the algorithm above. The estimated time should be in a <div> element with the content "Please wait around <minutes_to_wait> minutes" where <minutes_to_wait> is the number of estimated minutes:

<div>Your number is 7</div>
<div>Please wait around 49 minutes</div>

To store the data, you can use Python dictionaries and lists. For example, you could create a dictionary called line_of_cars with keys matching the service names and values that are the lists of customer tickets.

Examples

If you have two customers that require an oil change, three clients waiting for their tires to be inflated, and one needing diagnostics, the estimated time for the next customer would be one of the following:

  • To change oil: 2 * 2 minutes = 4 minutes
  • To inflate tires: 2 * 2 minutes + 3 * 5 minutes = 19 minutes
  • To get diagnostics: 2 * 2 minutes + 3 * 5 minutes + 1 * 30 minutes = 49 minutes
